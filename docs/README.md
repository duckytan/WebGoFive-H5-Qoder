## 🎯 文档阅读指南

### 🚀 新手开发者
1. 先阅读 [项目总结](./overview/PROJECT_SUMMARY.md) 了解项目整体
2. 阅读 [新项目开发指南](./guides/新项目开发指南.md) 了解技术架构
3. 查看 [开发者快速参考](./reference/开发者快速参考.md) 开始编码
4. 参考 [项目规范与安全指南](./guides/项目规范与安全指南.md) 了解规范

### 👨‍💻 经验开发者
1. 快速浏览 [开发者快速参考](./reference/开发者快速参考.md)
2. 查阅 [问题排查指南](./reference/troubleshooting-guide.md) 了解已知问题
3. 参考 [开发者指南](./guides/developer-guide.md) 了解架构细节
4. 需要时查询 [API文档](./reference/api-documentation.md)

### 🎮 算法开发者
1. 重点阅读 [五子棋规则与算法](./design/五子棋规则与算法.md)
2. 深入研究 [AI难度等级设计](./design/AI难度等级设计.md)
3. 参考 [活三形态参考](./reference/live-three-patterns.md)

### 📋 产品经理
1. 阅读 [需求规格书](./requirements/需求规格书.md) 了解需求
2. 查看 [用户使用指南](./user/user-guide.md) 了解用户体验
3. 参考 [项目总结](./overview/PROJECT_SUMMARY.md) 跟踪进度

### 🎨 设计师
1. 阅读 [界面设计](./design/INTERFACE_DESIGN.md)
2. 参考 [用户使用指南](./user/user-guide.md)
3. 了解 [需求规格书](./requirements/需求规格书.md) 中的UI要求

## 📝 文档维护规范

### 文档分类说明
- **Overview**: 项目概览、进度跟踪、任务管理
- **Requirements**: 需求定义、验收标准
- **Design**: 架构设计、算法设计、UI设计
- **Guides**: 开发规范、最佳实践、教程
- **Reference**: API文档、技术参考、速查手册
- **User**: 用户手册、使用指南
- **Operations**: 部署、运维、维护相关
- **History**: 历史记录、版本日志、开发总结

### 更新原则
- **实时更新**: Reference (API文档、快速参考)
- **版本同步**: Guides (开发指南)、Requirements (需求文档)
- **稳定文档**: Design (规则设计) - 基本不变
- **归档文档**: History (历史文档) - 只增不改
